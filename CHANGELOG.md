## [Unreleased]

### Исправлено

- **Сырой markdown в подсказках визарда**: экспандеры на шаге «Select data
  about your candidate users» (и блок «What is this and when to use it» на
  странице Validation) показывали литеральные `**` и дефисы-списки вместо
  форматирования. Теперь рендерятся через тот же `react-markdown`, что и
  остальные подсказки (`HelpCollapse`/`MarkdownBlockView`) — без новой
  зависимости.

### Изменено

- **Icon-only кнопки Bulk select / Import / Edit**: подписи убраны, смысл
  несут tooltip + `aria-label` (последний совпадает с прежним текстом, так
  что e2e-селекторы по имени продолжают работать). Bulk select — на обеих
  страницах со списком; Import — в шапке списка тестов; Edit — в шапке
  страницы теста.
- **«⋯» в шапке страницы теста — крайним справа**: overflow-меню по конвенции
  идёт последним; было слева от Edit, стало справа, меню выровнено по правому
  краю (`bottomRight`).
- **Панель папок свернута по умолчанию, выбор запоминается**: раньше она
  всегда открывалась развернутой и забывала выбор при уходе со страницы
  (локальный `useState`). Теперь стартует свернутой, а развернул/свернул —
  сохраняется НА СЕРВЕРЕ per-user (`users.folders_panel_collapsed`, миграция
  0018, аддитивная) и переживает навигацию, перезагрузку и повторный вход.
  Первая пользовательская настройка в приложении: `PATCH
  /auth/me/preferences`, значение едет в `UserOut` вместе с `/login` и `/me`,
  поэтому лишнего запроса на старте нет.

### Добавлено

- **Share: ссылка на тест в «⋯»** — копирует permalink и, если тест черновик,
  сразу предупреждает, что открыть его смогут только владелец, явно
  приглашенные и админы. Доступна любой роли, которая видит тест (включая
  viewer). Ссылка id-шная и переживает переименование теста, в отличие от
  адресной строки: для этого добавлены `ExperimentDetail.id`, `GET
  /experiments/by-id/{id}` и фронтовый роут-редирект — узкий permalink-вход,
  а не отложенная миграция адресации на uuid. Копирование работает и на
  обычном http (fallback на execCommand, где `navigator.clipboard`
  недоступен).

- **Экспорт/импорт эксперимента zip-архивом**: тест выгружается одним
  `<name>_export.zip` (дизайн, назначения, блоки, теги, статусы, все прогоны
  анализа, отчеты) и импортируется обратно — в тот же инстанс (клон) или в
  другой (миграция). Экспорт — Editor+ на любой ВИДИМЫЙ тест (владения не
  требует, экспорт — чтение): кнопка в hover-действиях списка и в «⋯» на
  странице теста. Датасеты по умолчанию идут ссылкой (имя + sha256
  содержимого), галочка «Include dataset snapshots» вкладывает сами данные для
  переноса между инстансами. Импорт всегда создает НОВЫЙ черновик, владелец —
  импортирующий, конфликт имени → `<name> (imported)`; ссылки на датасеты
  разрешаются в порядке sha256 → имя (с подтверждением) → снапшот → импорт с
  предупреждением «датасет не найден, переанализ недоступен до перепривязки».
  Архив новее поддерживаемого формата отклоняется с внятным сообщением. Оба
  события — в audit_log.
- **Глобальная кнопка «+» в шапке** (Superset-style, Editor+): «A/B test» →
  визард дизайна, «Dataset» → создание датасета, из любого места приложения.
  Существующие кнопки создания на страницах остались — это дополнительный
  вход, не замена.

## [2.5.1] — 2026-07-17

Follow-up to v2.5.0's own item 8 — first real test of the new "tag
regularly" rule (CLAUDE.md), one small package, one patch tag.

### Исправлено

- **Version display, second pass**: v2.5.0 read `ABKIT_VERSION` (a build-arg
  turned env var) as the primary source, which stripped the tag's `v`
  prefix and showed a bare `dev (<sha>)` for any local build made after a
  tag, with no indication of how far past the release it was. Now the
  single source is `git describe --tags --always --long`, computed once at
  image-build time into `/app/VERSION_DESCRIBE`: the tagged commit itself
  shows `vX.Y.Z`; N commits past a tag show `vX.Y.Z+N (<sha>)`; no tag at
  all shows `dev (<sha>)`. CI's `build-and-push` checkout now uses
  `fetch-depth: 0` so the tag is resolvable during the image build.

### Изменено

- **"Uncategorized" в панели папок**: не папка (нет rename/delete) —
  перемещена в конец списка (после пользовательских папок), показывается
  только если что-то реально не разложено по папкам, и всегда в
  приглушенном стиле, чтобы читаться как системный вид, а не созданная
  папка.

## [2.5.0] — 2026-07-16

75 коммитов со времени v2.0.0 (2026-07-08) — крупнейший набор правок между
тегами на сегодня, отсюда и минорный, не патч-номер. Породил «Правило:
релизный процесс» ниже: version на About/в отчетах был все это время
заморожен на 2.0.0 (item 8 итогового пакета) именно из-за отсутствия
промежуточных тегов — регулярные теги с этого релиза обязательны.

### Добавлено

- **Database Connections + датасет-центричная модель** (DB1-DB5): подключения
  к внешним БД (PostgreSQL, ClickHouse, MSSQL), пароль шифруется Fernet;
  датасет можно материализовать из `SELECT`-запроса (SELECT-only guard,
  стриминг чанками в parquet, лимиты `ABKIT_SQL_MAX_ROWS`/
  `ABKIT_SQL_TIMEOUT_SEC`); страница **Датасеты** — единственное место
  загрузки файлов/создания SQL-датасетов, дизайн/анализ/валидация выбирают
  из существующих датасетов вместо прямой загрузки.
- **Schema/Table browser**: каскадные selects при создании датасета из SQL
  (интроспекция схем/таблиц подключения, кэш 60с), автоподстановка
  `SELECT * FROM "schema"."table"`, явное сохранение выбора
  (`source_schema`/`source_table`) вместо повторного парсинга SQL при
  каждом открытии Edit.
- **Edit/Delete датасета**: переименование и правка запроса/подключения для
  `source=sql` (переиспользует механизм Refresh), удаление с проверкой
  использования (простое подтверждение для неиспользуемых, ввод `DELETE`
  для используемых экспериментами) — без владельца/Admin недоступно.
  Bulk select/delete датасетов (одно подтверждение на весь батч).
- **Data preview в Edit**: превью сохраненного снапшота и live-превью
  результата еще не сохраненного запроса (сравнение бок о бок).
- **Живой поиск и фильтр по Source** на странице Датасетов (без Enter).
- **Refresh датасета из UI**: перевыполнение сохраненного SQL с сохранением
  атомарности (обрыв соединения не портит существующий снапшот).
- **Теги для A/B тестов** (по образцу Superset): свободная группировка,
  typeahead-создание из Edit Properties, фильтр по нескольким тегам (AND),
  поиск по имени тега, страница управления (rename/merge/delete, в т.ч.
  bulk) для Admin.
- **Папки для A/B тестов**: одноуровневая группировка (без вложенности),
  коллапсируемая панель слева от списка с счетчиками, move — построчно и
  bulk, удаление папки не удаляет тесты (возврат в Uncategorized).
- **Design-визард — существенная переработка**: sample-size-first флоу с
  жестким Next-гейтом, явный выбор метода анализа (мультиселект вместо
  Advanced options), редактируемые alpha/power, опциональное поле
  Hypothesis, шаг переименования датасета/колонок при загрузке CSV, проверка
  мощности по стратам, отдельные кнопки скачивания CSV по группам, External
  split mode (для Firebase-подобных внешних сплитов).
- **MDE/Results таблицы**: округление до 3 знаков, бинарный baseline в
  процентах, честный отдельный MDE для вторичных метрик (не копия
  первичного), обязательное округление вверх (ceil) для требуемого n.
- **Variant flow images**: скриншоты флоу по группам с drag&drop
  переупорядочиванием (dnd-kit), лайтбокс на Design-вкладке, встраиваются в
  design-отчет.
- **Group descriptions, lifecycle dates, chart tooltips** везде на графиках;
  **per-dimension segment analysis** в Results и HTML-отчете;
  **distribution charts**: positive-only режим отображения + фикс клиппинга
  P99.
- **Analysis UX**: явный запуск анализа (не автозапуск при подготовке
  данных), пере-запуск с новым датасетом, per-metric вид аналитики,
  дедупликация строк результатов, колонка CUPED ρ в детальной таблице.
- **Validation UX**: auto-datasource (датасет дизайна подставляется
  автоматически, без повторной ручной загрузки).
- **Admin Monitoring** (item 6 итогового пакета — своя страница
  `/settings/monitoring`, изначально была табом в List Users): память
  backend-процесса, размер БД/данных на диске, свободное место, история с
  графиками (raw 24ч + часовые агрегаты 90 дней), топ-10 таблиц по размеру.
- **Bulk select Deactivate/Activate в Admin → Users** (item 7): не delete —
  самозащита (нельзя деактивировать себя или последнего активного admin).
- **Аудит-лог с деталями изменений** (item 4): History-таб и глобальный Audit
  теперь показывают ЧТО изменилось, не только факт изменения ("status:
  designed → running", "tags: +checkout, −q2" и т.п.), с раскрытием для
  сложных диффов.
- **Superset-style реестр экспериментов**: аватары владельцев, относительное
  время, hover-reveal действия, bulk select/delete, Edit Properties modal
  (доп. владельцы/редакторы, видимость по ролям, теги).
- **Unsaved-changes guard + контракт свежести кэша после мутаций** —
  системная инфраструктура (см. CLAUDE.md), а не точечная фича: единый хук на
  все формы/модалки/визард, централизованный реестр query-ключей.
- Полностью английский UI + брендинг **ABKit** (логотип-кит, favicon,
  HTML-отчеты).
- **Ребрендинг: ABKit → ABSet.** Новое видимое имя продукта и логотип везде
  (навбар, страница логина, вкладка браузера, favicon, Settings → About,
  шапки HTML-отчетов, CLI-тексты, вся документация). Технические
  идентификаторы (python-пакет `abkit`, пути `/data`, docker-сервисы,
  env-префикс `ABKIT_*`, имя БД/пользователя postgres, CLI-команда
  `abkit-admin`) осознанно не переименованы — см. CLAUDE.md. Репозиторий на
  GitHub переименован `abkit` → `abset` (редирект со старого имени работает).

### Исправлено

- **Version на About/в отчетах была заморожена на 2.0.0** (item 8 итогового
  пакета) — теги `vX.Y.Z` никогда не транслировались в собранный образ
  (`ARG` был только OCI-лейблом, не читался приложением); теперь единый
  источник — git-тег (см. новое правило ниже), локальный билд без явного тега
  показывает `dev (<short sha>)`.
- **Клик в поле "type DELETE" открывал drawer превью строки под модалом**
  (item 3 итогового пакета) — React-portal bubbling через `onRow` таблицы;
  системный фикс (`StopClickPropagation`) применен ко всем delete/edit/bulk
  модалам над таблицами, не точечно.
- Убраны два шумных предупреждения на Design-вкладке (MDE > baseline,
  недобор группы) — добавленные в предыдущем пакете, но признанные шумными
  по фидбеку с реального использования.
- Гонка/крэш job'ы Compare alternative methods + человекочитаемые сообщения
  об ошибках джобов; восстановление прерванных (interrupted) джобов при
  рестарте backend; снижение waterline RSS backend после тяжелых джобов +
  `mem_limit` страховка.
- Приведение типов `unit_id` при джойне пре/пост-периода (падало на
  несовпадающих dtype), человекочитаемые ошибки вместо трейсбека; CUPED
  больше не падает, если заявленная pre-колонка отсутствует в пост-данных.
- History-таб фильтровался по имени эксперимента, а не id — путал события
  удаленного и заново созданного под тем же именем эксперимента.
- Скачивание samples с не-ASCII (кириллица) в имени эксперимента падало на
  Content-Disposition; `error_id` в внутренних ошибках джобов для трассировки
  по логам.
- `index.css` никогда не импортировался — hover-reveal действия и
  выделение designed-строк молча не работали.

## [2.0.0] — 2026-07-08

**React-UI + FastAPI backend — миграция с Streamlit завершена (этапы R1-R8,
FRONTEND.md).** Крупнейший релиз проекта: полностью новый интерфейс,
Streamlit (`app.py`) и сервис `legacy` удалены безвозвратно.

### Добавлено

- **React-UI** в стиле Apache Superset: реестр экспериментов, мастер
  дизайна (выбор данных/групп/метрик/страт/MDE в шагах), страницы Analyze
  (таблица результатов, forest/distribution/segment/cumulative-графики) и
  Validation (A/A/A/B симуляции), управление пользователями и аудит-лог
  (роль Admin).
- **FastAPI backend** поверх существующего ядра `abkit/`: REST API,
  cookie-аутентификация, Alembic-миграции и bootstrap первого
  администратора при старте, джоб-раннер (ThreadPoolExecutor) для
  design/analyze/validate с прогрессом и человекочитаемыми ошибками.
- **Docker-развертывание в 4 сервиса**: `nginx` (единая точка входа,
  маршрутизация `/` → frontend, `/api/*` → backend), `frontend` (статика),
  `backend`, `postgres`. Публикация образов в `ghcr.io` на тег `v*`
  (`.github/workflows/ci.yml`, джоба `build-and-push`).
- Принудительная смена временного пароля при первом входе, self-registration
  (опционально, роль Viewer по умолчанию, как в Apache Superset).
- Полный набор автотестов React-UI: `backend/tests/` (pytest) +
  `frontend/` (typecheck/lint/build) + Playwright e2e против реального
  docker-compose стека.

### Убрано

- Streamlit-интерфейс (`app.py`) и сервис `legacy` — полностью удалены,
  единственный интерфейс — React-UI. DESIGN.md/DOCKER.md сохраняют разделы
  про Streamlit только как историю технического задания.
